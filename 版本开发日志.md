# ycIDE 版本开发日志

> 用途：先记录开发变化，再在发版时据此更新 `版本更新说明.md`，避免遗漏。

## 记录规范
- 每次功能开发/修复/优化完成后，追加到“待发版条目”。
- 分类只使用：`[新增]`、`[修复]`、`[优化]`。
- 每条尽量包含：变化摘要、影响范围、验证方式。
- 版本号统一使用 `x.y.z-beta.n` 预发布格式（示例：`0.0.3-beta.54`），不再使用 `0.0.3.54` 这类纯数字补丁格式。
- 发版改版本号时，需同步修改：`src/renderer/src/components/StatusBar/StatusBar.tsx` 与 `src/renderer/src/App.tsx`（`aiIdeContext` 版本字符串）。
- 发版后将对应条目移动到“已发布归档”。
