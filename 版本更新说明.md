# ycIDE 版本更新说明

## ycIDE - 开发测试版 v0.0.3-beta.54(2026-05-09)

---

### 新增功能

- [新增]项目树支持项目级右键菜单：在项目根节点可单独执行“保存该项目全部文件”“关闭该项目”，用于多项目并行时按项目粒度管理。

### 问题修复

- [修复]修复 Icon 组件在 Problems 面板中的 inline style 告警：移除 JSX `style` 属性，改为通过 ref 在运行时设置宽高，保持 `size` 行为不变。
- [修复]修复项目树双击文件名无法打开文件：文件节点打开路径改为优先使用真实文件名（含扩展名），并统一“模块/窗口/资源”文件节点的双击与回车打开行为。
- [修复]修复粘贴代码产生重复子程序：当粘贴内容中的子程序名已存在于当前文件时，不再插入重复子程序头，改为将该子程序体内容追加到现有子程序末尾。
- [修复]新增重复子程序诊断：手工输入同名 `.子程序` 时，问题面板会报“子程序重复定义”错误。
- [修复]清理 VS Code Problems 面板中的 OutputPanel/AIAssistantPanel/VisualDesigner 报错：移除内联样式、补齐菜单与表单可访问性属性、改为 CSS 变量驱动动态尺寸与定位，并补齐 Safari 的 `-webkit-user-select` 兼容前缀。
- [修复]修复 AI 模型配置持久化被覆盖：模型配置弹窗保存后同步更新 App 内存设置（模型/DeepSeek Key/GLM Key/自定义模型），避免后续保存其它设置时将已保存 Key 覆盖回空值。
- [修复]修复 Agent 模式错误提示显示位置：当执行失败（如 API Key 缺失）时，错误信息优先追加到 AI 返回区，不再只显示在输入框下方状态栏。
- [修复]修复 AI 面板点击发送后偶发“无返回”问题：为聊天/流式聊天 IPC 调用补充异常兜底，发生调用异常时返回明确错误而非卡住等待。
- [修复]修复主进程 `file:readDir` 对文件路径抛出 `ENOTDIR`：读取前新增目录类型校验并统一异常降级为空列表，避免日志刷屏及上层调用被异常打断。
- [修复]修复多项目打开时项目树被后开项目覆盖：项目根节点改为按项目目录生成唯一 ID，打开/新建项目时按末尾追加（同项目则替换原根），刷新项目树仅替换当前项目对应根节点。
- [修复]修复可视化设计器编辑后项目树未实时显示未保存标记：`handleFormChange` 增加 `onOpenTabsChange` 同步，确保 Sidebar 使用到最新 `.efw` 标签内存状态计算 dirty。
- [修复]修复项目树未显示窗口/源码文件未保存标记：项目树 dirty 匹配键改为优先使用节点文件 ID（含扩展名），不再误用显示标签（无扩展名），使 `.efw/.eyc/...` 的“未保存圆点”能正确显示在右侧边框内侧。
- [修复]修复表格模式流程线在 `如果/如果真` 连续块中的误连通：`如果结束/如果真结束` 后紧跟 `如果/如果真` 时，不再套用“判断”专用的上下双横线桥接效果。
- [修复]修复资源管理器双击子程序偶发跳错行：编辑导致行号漂移后，项目树双击子程序改为按子程序名实时定位（失败时回退旧行号），不再闪到新输入代码行。
- [修复]修复删除子程序后项目树残留旧节点：编辑器内容变更会实时同步 `.eyc` 子程序节点到项目树，未保存状态下删除 `.子程序` 也会立即从“程序集”树中消失。
- [修复]修复属性面板事件下拉勾选状态不刷新：事件列表“✓ 已存在子程序”改为优先读取当前打开标签页内存内容，并用最新解析结果覆盖缓存；删除子程序后勾选会即时取消。
- [修复]修复项目树双击子程序后落点行为：闪烁定位保持在子程序声明行不变，同时将输入光标置于该子程序首个普通代码行的行首（如子程序在第17行、首行代码在第18行则光标落到18行首）。
- [修复]修复项目树双击子程序后“仅闪烁无光标”问题：将光标进入时机调整为闪烁定位之后并增加重试聚焦，避免滚动重排冲掉编辑态；当子程序内仅空白普通行时也可在行首显示光标。
- [修复]修复双击子程序后输入框未渲染导致“仍看不到光标”：纠正 `startEditLine` 调用中的 `isVirtual` 参数，避免与普通代码行渲染条件不匹配而隐藏编辑输入框。
- [修复]修复子程序内“空行优先”落点：当子程序声明后先有若干空行再有命令时，双击项目树后光标改为落在首条空普通行行首，不再跳到后续命令行。
- [修复]修复可视化设计器创建子程序后的“无行号空行”问题：新建子程序时直接生成真实空普通行；项目树跳转到无普通行子程序时会自动物化一条空普通行并聚焦行首，避免必须手动回车后才正常。
- [修复]统一“已存在子程序”导航逻辑：属性面板事件下拉选择与可视化设计器双击组件进入默认子程序时，行为改为与项目树双击一致（子程序声明行高亮闪烁 + 光标定位到子程序首个普通行首列，并在缺失普通行时自动物化空普通行）。
- [修复]修复子程序跳转抖动与多点闪烁：去除 App 层对子程序导航的多重重试调用，并在编辑器侧为子程序导航增加“最后一次生效”的序列去重，改为单次闪烁 + 单次聚焦，避免上下抖动与多个位置依次高亮。
- [修复]修复“组件双击/属性面板事件下拉”跳转闪烁偏位：将 `navigateOrCreateSub` 的“已存在子程序”分支改为复用 `navigateToSubprogram` 同一内部实现，统一三入口（项目树/组件双击/事件下拉）的定位与闪烁规则，并补齐子程序名 trim 匹配。
- [修复]修复子程序声明行闪烁偶发错位：统一跳转内部逻辑新增“优先行号命中同名子程序”策略（组件双击/事件下拉传入已知声明行），并在新闪烁前清理旧 `highlight-flash` 样式，避免残留闪烁造成错位错觉。
- [修复]修复子程序跳转后“先在下方后瞬间居中”导致的闪烁位置偏差：声明行定位由 `center` 调整为 `nearest`，并移除光标聚焦阶段的额外滚动，避免二次跳动造成闪烁观感错位。
- [修复]修复子程序跳转时“滚动未稳定先闪烁”导致的错位：声明行高亮改为在两帧 + 短延时后触发（滚动收敛后再闪烁），并继续受导航序列去重保护，避免瞬态位置上闪烁。
- [修复]修复顶部菜单“插入 -> 子程序”后仅出现虚拟空行的问题：插入结构改为“真实空行 + 子程序声明 + 真实空普通行”，并在插入后将光标置于新子程序首条普通行行首，避免必须手动回车物化。
- [修复]新增子程序“保底普通空行”删除规则：子程序首条普通行始终保留为可输入占位；删除命中该行时，若有内容则改为空行保留，若本身为空则改删其下方普通行；多选删除/剪切同样遵循该规则，避免全删后无法进入普通代码行。

### 优化调整

- [优化]优化 AI 面板停止按钮视觉：移除停止按钮红色外边框，仅保留错误语义色图标与背景高亮，避免边框过重。
- [优化]调整标题栏拖拽区域设置方式：移除 CSS 中的区域声明，改为在运行时通过 `-webkit-app-region` 写入拖拽/禁拖拽属性，降低 Problems 面板兼容性提示噪音并保持 Electron 标题栏行为。
- [优化]优化 AI 面板模式下拉宽度：加宽模式选择框，避免选择 Agent 时文本末尾被遮挡。
- [优化]项目树未保存标记支持分类联动：当“窗口/程序集/全局变量/常量表/自定义数据类型/DLL命令/资源”等分类下任一文件存在未保存修改时，分类节点也显示右侧未保存圆点。

- [修复]修复 `.e/.ec` 导入只生成“导入说明”或空子程序占位的问题，导入后可生成真实 `EYC/EGV/EDT/ELL/ERC` 内容。
- [修复]修复核心命令号映射偏差：补齐 `Lib0` 回退映射并覆盖 `181/182/203` 兼容点，减少 `_Lib0CmdXXX` 占位显示。
- [修复]修复非核心支持库命令映射缺失：补充多支持库命令 ID 映射，减少 `_LibXCmdY` 回退。
- [修复]修复特殊结构下（`判断开始` 与后续 `判断` 分离）内侧线缺失、断段与贯穿不完整问题，补齐上一行起始横线及跨行内侧竖线连通。
- [修复]修复“第 7 行下三角箭头残留”系列问题，统一屏蔽重叠分支下不应显示的下箭头。
- [修复]修复内侧线“上半段显示规则”异常：仅对“从上方延续”的行显示上半段，避免首行误绘。
- [修复]修复 `如果真` 结束箭头在隐藏结束行后的显示异常：补回向下三角并去除多余横向虚线。
- [修复]修复嵌套 `如果真` 场景中结束箭头锚定错层与漏显问题，改为按同深度独立锚定与绘制。
- [修复]修复手工输入 `.如果` 回车未补齐可见 `否则/如果结束` 的问题，并修复文本/表格模式可见性不一致。
- [修复]修复表格编辑器偶发“临时回退文本模式”与编辑中空白闪烁问题。
- [修复]修复并清理 VS Code Problems 面板剩余告警（ARIA、inline style、类型收窄、不兼容 CSS）。

### 优化调整

- [优化]流程线与流程命令解析进一步收敛为“命令驱动”路径，去除旧 marker 兼容分支的生效逻辑，降低流程缩进与连线异常来源。
- [优化]表格模式性能系统性优化：可视区 block/行级裁切、滚动节流、缓存复用、minimap 阈值收紧，降低大文件卡顿。
- [优化]将语法诊断与表格模型构建迁移到 Worker，减少主线程阻塞与首开空白时间。
- [优化]补强“判断自动展开”诊断日志，新增关键判定上下文，便于定位第 5/7 行结构未命中原因。
- [优化]调整首屏渲染策略：先渲染轻量前置行，再异步切换完整模型，改善大文件打开体验。
