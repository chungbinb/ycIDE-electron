# ycIDE 版本更新说明

## ycIDE - 开发测试版 v0.0.3-beta.55(2026-05-15)

---

### 新增功能

- [新增] 新增内置数据类型“大整数型”：编译后端引入 `YC_BIG` 任意精度整数实现，支持 `+ - * /` 与比较运算，并支持超出长整数范围字面量在大数上下文保真计算。
- [新增] 编辑期类型诊断增强：当文本型变量直接接收数值型右值（如 `e = a + b`、`e = a`，且右值为数值型）时，问题面板立即报错并触发行内红色波浪线。
- [新增] 编辑期问题面板新增“长整数常量超范围”诊断：在代码书写阶段实时检查整数常量，超出 `[-9223372036854775808, 9223372036854775807]` 立即报错，无需等待编译。
- [新增] `spec` 支持库最小后端协议映射：新增 `spec.protocol.json`，优先在编译期直连 `Trace/Verify/Delay`（分别对应输出调试、检查、中断延时）。
- [新增] 新增系统特殊功能支持库 `spec`（独立于核心支持库）：基于开发期外部 `D:\chungbin\ycIDE-html\支持库源码\spec` 命令定义离线生成项目内支持库清单，运行时仅读取 `lib/spec` 产物。
- [新增] 新建子程序自动补局部变量：在新子程序声明下方自动插入 `.局部变量 局_变量N, 整数型`，变量名按 `局_变量1` 递增并避免重名。
- [新增] EYC 表格编辑器右键菜单支持定制化分组与动作：新增“新子程序/删除行/插入新行/编译当前行/屏蔽/解除屏蔽”菜单项，并按要求调整分隔顺序与快捷键文案展示。
- [新增] 状态栏右下角新增当前活动文件编码显示，并支持点击后选择编码重新打开：新增编码菜单（UTF-8/UTF-8 BOM/GB18030/GBK/Big5/Shift_JIS/UTF-16LE/UTF-16BE/Windows-1252），选择后按指定编码重新读取当前文件。
- [新增] 新增“打开文件夹工作区”入口：在保留现有“打开项目”流程不变的前提下，新增独立菜单/快捷键入口用于选择文件夹；若根目录存在 `.epp` 自动按项目打开，否则按普通文件夹工作区打开并显示目录树。
- [新增] 核心支持库 ABI 契约进入实施版 v1：在 `windows.cpp` 落地 OLE Automation Date 适配基础，并新增 `ToTime/now/GetDatePart/GetTimePart` 真实实现（移出自动桩区）；同时将契约与映射规范升级到 v1，映射产物新增 `abiSpecVersion` 与 `abiContractRef` 字段用于高风险命令追踪。
- [新增] 启动核心支持库 ABI 约定与命令映射实现：新增 `krnln-abi-contract.md` 与 `krnln-command-mapping.md` 讨论版规范，并落地自动映射脚本 `scripts/krnln/generate-command-mapping.mjs`，可生成 `command-mapping.windows.json` 与映射报告，首轮统计 `total=512 / implemented=133 / stubbed=379 / undeclared=0`。
- [新增] 一次性补齐核心支持库 Windows 全量导出符号层：根据 `krnln.commands.ycmd.json` 自动生成并落地 443 个缺失 `krnln_*` 导出桩函数，统一按返回类型给默认返回值，避免编译/链接阶段缺符号报错。
- [新增] 补全核心支持库 Windows 磁盘命令实现：新增 `ChDrive/GetDiskTotalSpace/GetDiskFreeSpace/GetDiskLabel/SetDiskLabel`，支持按驱动器查询容量与卷标，并可切换当前驱动器。
- [新增] 补全核心支持库 Windows 文件系统常用命令实现：新增 `CurDir/ChDir/MkDir/RmDir/FileCopy/FileMove/kill/name/IsFileExist/FileLen/GetAttr/SetAttr/GetTempFileName` 平台实现，覆盖目录切换、递归建删目录、文件复制移动删除、属性读写与临时文件名生成。
- [新增] 完善核心支持库跨平台基础实现：`linux.cpp` 与 `macos.mm` 从占位实现扩展到可用基础命令，实现消息框/鸣叫/延时/运行命令/取执行文件与运行目录/环境变量读写/命令行读取/系统版本与语言等通用入口。
- [新增] 新增误报申诉包收集脚本：支持收集样本路径、SHA-256、构建上下文（编译器/目标/模式）与附加日志，统一生成 `debug_logs/av-report-*/report.json` 与附件目录。
- [新增] 新增 Windows 签名打包入口：增加 `package:win:signed`，通过环境变量注入证书参数（含强制签名开关），为发布链路接入代码签名提供落地脚本。
- [新增] 核心支持库 Windows 平台实现扩容：在 `impl/windows.cpp` 批量落地系统命令与组件基础操作（消息框、蜂鸣、延时、取屏幕/鼠标、环境变量、剪贴板、运行命令、取程序路径、窗口创建/销毁/激活/焦点/尺寸/文本/重画等），并增加命令命名兼容入口。
- [新增] 核心支持库 Windows 平台实现起步：`impl/windows.cpp` 的 `krnln_message_box` 从占位返回改为真实调用 WinAPI `MessageBoxA`，并补充空文本/空标题入参兜底，打通首个可用平台命令实现。
- [新增] 支持库管理接入在线索引和独立 zip 安装：默认读取 `https://ycide.dev/libraries/index.json`，按支持库 ID 下载独立包，校验 SHA-256 后安装到用户数据目录，并在支持库管理窗口显示下载、安装、加载、可更新和失败状态。
- [新增] 系统设置新增支持库在线索引地址配置，便于后续切换官网源或镜像源。
- [新增] 新增支持库发布打包脚本，可将 `lib/<id>` 生成 `packages/<id>.zip` 和 `index.json`，适配官网独立支持库包部署结构。

### 问题修复

- [修复] 编辑框运行态默认文本与设计器对齐：编译生成控件时，编辑框/超级编辑框/文本框不再将控件名回退为初始文本；仅在显式设置文本时才显示内容，避免设计器为空而运行后出现默认文字。
- [修复] 可视化设计器编辑框默认底色与运行态对齐：编辑框/超级编辑框预览改为优先读取控件背景属性，未设置时默认白底，避免深色主题下显示为黑色而与编译运行结果不一致。
- [修复] 参数递归展开子级箭头层级偏移修正：子级箭头定位显式叠加当前层缩进变量，箭头锚点改为按“本层起始竖线”计算，避免误落到上一级位置。
- [修复] 参数递归展开子级箭头绘制基准修正：子级箭头改为在参数子级容器顶部绘制（不再悬挂在上一行底部重叠区域），减少跨行基准导致的对齐漂移。
- [修复] 参数递归展开子级箭头中心线对齐：子级箭头改为按中心点锚定下一级竖线起点（`translateX(-50%)`），避免按左边界定位导致“看起来未对齐”的偏移。
- [修复] 参数递归展开子级箭头锚点修正：箭头改为锚定下一级竖线起点位置，不再对齐到父级收口结束点。
- [修复] 参数递归展开子级连线对齐修正：二级展开三角箭头与该层竖线中心线重新对齐，消除子级箭头与竖线左右错位。
- [修复] 参数递归展开连线收口修正：当某参数行继续展开下级子树时，父级该行的竖线在本行中线收口，避免出现“应为 L 形却显示为 ├”的错连线。
- [修复] 消除文本字面量赋值告警：文本型变量接收 `L"..."` 字面量时改为走 `yc_value_to_text(...)` 复制，避免 `wchar_t*` 直接绑定字符串常量触发 `-Wwritable-strings`。
- [修复] 工具栏图标按钮偶发需点击两次才响应：鼠标按下时立即执行按钮动作，并对后续 click 做去重，减少首次点击被焦点/上下文清理吞掉的情况，同时保留键盘触发能力。
- [修复] 编辑期长整数溢出诊断与大数类型对齐：当赋值目标变量为“大数”时，不再错误提示“超出长整数范围”。
- [修复] 编译后端文本变量拼接语义修复：表达式翻译新增变量类型感知，`文本型 + 文本型` 生成 `yc_text_concat(...)`，避免误生成为 `wchar_t*` 指针相加导致编译错误（如 `e = t1 + t2`）。
- [修复] EYC 文本模式编辑器接入 Monaco 诊断标记：问题面板诊断结果同步为编辑区红色波浪线（含错误/警告级别）。
- [修复] 生成代码编译失败收敛：`spec` 调试输出协议翻译补命令上下文（支持 `到文本(...)` 参与表达式转换），并修复“文本型变量接收数值表达式”时的类型不兼容（自动转换为文本）。
- [修复] 表达式翻译增加整数字面量 64 位边界钳制，避免超长整数直接触发 C++ “integer literal is too large” 编译错误。
- [修复] ycmd 后端接入校验补齐英文命令名匹配：`CommandSignatureDef` 保留 `englishName` 并在协议匹配时传入，修复仅配置 `commandEnglishName` 的协议被误判“未接入后端实现”（如 `spec.Trace`）。
- [修复] 编译后端支持 `repeatable` 参数封送：ycmd 泛型命令调用在参数超出固定声明长度时，正确复用末尾可重复参数的数据类型，不再错误回退为“整数型”。
- [修复] 系统特殊功能支持库 `spec` 可追加参数命令补充 `repeatable` 标记，覆盖 `Trace/Verify/CallFunction/DelayCallFunction/GetText/DelayCallFunction_precisely`。
- [修复] 系统特殊功能支持库 `spec` 命令参数元数据全量对齐外部源码：补齐 `MachineCode/GetVarAddress/GetSubAddress/GetVarDataAddr/Verify/CallFunction/Delay/DelayCallFunction/AllocMem/FreeMem/ESysFunction/GetText/GetText_Set/GetText_Get/DelayCallFunction_precisely` 的参数定义（名称、类型、可省略）。
- [修复] 系统特殊功能支持库 `spec.Trace` 参数元数据补齐：`调试输出` 命令从“无参数”修正为首个可省略参数“欲输出值（通用型）”，与外部源定义对齐，恢复调用提示可读性。

- [修复] 可视化设计器组件箱两种显示模式的宽度联动修正：图标模式移除固定宽度锁死，支持停靠态拖拽调宽；列表/图标模式下组件项图标、字号、间距与卡片尺寸会随组件箱宽度同步缩放，不再固定大小显示。
- [修复] 第三方支持库组件缺失 `icon` 字段时图标兜底修正：改为统一显示“未知”图标，不再按组件中文名误回退到图片等其他图标；仅核心支持库保留名称映射兜底。

### 优化调整

- [优化] 参数递归展开接入流程层级配色：参数区连线与箭头改为复用流程命令的层级颜色算法（`resolveFlowLineColors`），多级参数展开时可按层级显示彩色线条。
- [优化] 命令参数展开支持表达式递归查看：代码行参数值现可继续展开 `+` 表达式（显示“被加数或文本或字节集 / 加数或文本或字节集”），并可进一步展开嵌套命令参数（如 `到文本(d)` 展示“待转换的数据：d”）。
- [优化] “正在生成C++代码”阶段提速与可观测性增强：项目元数据（全局变量/常量/资源/子程序/数据类型/DLL 命令）改为按项目指纹缓存复用，避免每次重复全量扫描；同时新增元数据阶段耗时日志。
- [优化] 运行阶段进一步提速：新增可执行产物指纹缓存（源 `.cpp`、资源、静态库、目标平台/架构/模式等），命中时直接跳过编译与链接，复用上次 `exe`。
